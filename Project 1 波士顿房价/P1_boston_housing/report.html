编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    
    score = r2_score(y_true, y_predict)

    return score

# 不允许导入任何计算决定系数的库

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    y_bar = np.mean(y_true)
    SS_res = 0
    SS_tot = 0
    for i in range(0 , len(y_true)):  
        SS_res += (y_true[i] - y_predict[i])**2
        SS_tot += (y_true[i] - y_bar)**2
    score = 1- (SS_res / SS_tot)

    return score

问题 8 - 交叉验证¶

  • 什么是K折交叉验证法（k-fold cross-validation）？
  • GridSearchCV是如何结合交叉验证来完成对最佳参数组合的选择的？
  • GridSearchCV中的'cv_results_'属性能告诉我们什么？
  • 网格搜索时如果不使用交叉验证会有什么问题？交叉验证又是如何解决这个问题的？

提示： 在下面 fit_model函数最后加入 print pd.DataFrame(grid.cv_results_) 可以帮你查看更多信息。

问题 8 - 回答：¶

  • k-fold cross-validation会将数据计划分为k份，1份作为验证集，另外k-1份作为训练集，由此我们就有了k组训练集/验证集的组合，就可以用某一参数组合进行k次训练和验证，然后对这k个验证的结果取均值，得到当前参数组合训练出的模型的整体评估；相比于train_test_split方法，通过k-fold cross-validation，可以在很大程度上避免由于运气不好而得到诡异划分造成的偏差，并且能对k次交叉验证的结果取均值，得到的评估结果更为客观。
  • GridSearchCV结合交叉验证：首先定义评估器：estimator；参数字典parameters；折数：kfold = KFold(n_splits=10)（假设分为10折）；分数：scoring_fnc = make_scorer(accuracy_score)（假设使用准确率作为评分依据），然后使用grid = GridSearchCV(estimator, parameters, scorning=scoring_fnc, cv=kfold)
  • GridSearchCV中的cv_results_可通过pd.DataFrame(grid.cv_results_).T转换成一个DataFrame，反映在每一种参数组合下，k折的训练分数和验证分数等参数，算出每一种参数组合下的平均测试分数和验证分数。由此可以找到最高的验证分数所对应的参数组合，从而确定这个参数组合为最优的。
  • 如果网格搜索时如果不使用交叉验证的话，每一种参数组合得出的分数是经过唯一的训练集/验证集计算出来的，具有很大的偶然性和不确定性。使用交叉验证就能避免这种偶然性，因为它将数据计划分为k份，1份作为验证集，另外k-1份作为训练集，由此我们就有了k组训练集/验证集的组合，就可以用某种参数组合进行k次训练和验证，然后对这k个验证的结果取均值，得到当前参数组合训练出的模型的整体评估，结果更具说服力更客观。
